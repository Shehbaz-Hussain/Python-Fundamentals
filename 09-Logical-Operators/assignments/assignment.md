# Logical Operators Assignments

## Introduction

This assignment set is designed to strengthen your understanding of Python logical operators through practical programming exercises. The assignments gradually increase in difficulty and only require the concepts learned up to **Module 09**.

These assignments are intended for self-study, classroom practice, laboratory work, and university assessments.

---

# Assignment Objectives

After completing these assignments, you should be able to:

- Understand the purpose of logical operators.
- Use `and`, `or`, and `not` correctly.
- Combine comparison and logical operators.
- Write simple decision-making programs using `if` statements.
- Evaluate Boolean expressions.
- Apply logical operators to solve real-world problems.
- Improve logical thinking and problem-solving skills.

---

- Complete all assignments using Python.

- Use meaningful variable names.

# Easy Assignments

## Assignment 1 — Age Eligibility Checker

Write a program that:

- Takes the user's age as input.
- Checks whether the user is **18 years or older**.
- Prints whether the user is eligible to vote.

---

## Assignment 2 — Number in Range

Write a program that:

- Takes a number as input.
- Checks whether it is between **10 and 50 (inclusive)**.
- Displays an appropriate message.

---

## Assignment 3 — Positive Even Number

Write a program that:

- Takes an integer as input.
- Checks whether it is **positive** and **even**.
- Display the result.

---

## Assignment 4 — Pass or Fail

Write a program that:

- Takes marks as input.
- Check whether the marks are **50 or greater**.
- Display whether the student has passed or failed.

---

## Assignment 5 — Weekend Temperature Check

Write a program that:

- Check whether the temperature is between **20°C and 30°C**.
- Display whether the weather is pleasant.

# Tips

- Solve one assignment at a time.
- Read the problem carefully before writing code.
- Break complex conditions into smaller comparisons.
- Use parentheses to improve readability when needed.
- Test your program with different input values.

---

# Best Practices

- Use meaningful variable names.
- Keep logical expressions simple.
- Write consistent indentation.
- Add comments to explain important parts of your program.
- Verify all conditions before printing the result.

---

# Common Mistakes

- Using `AND`, `OR`, or `NOT` instead of `and`, `or`, and `not`.
- Confusing comparison operators with logical operators.
- Forgetting to convert numeric input using `int()`.
- Writing unnecessarily complex logical expressions.
- Ignoring operator precedence.
