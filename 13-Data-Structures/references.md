# Python Enhancement Proposals (PEPs)

Although this module focuses on beginner-level concepts, the following PEPs provide useful background on Python's design philosophy and language evolution.

---

## PEP 8 – Style Guide for Python Code

https://peps.python.org/pep-0008/

Topics:

- Naming conventions
- Code layout
- Readability
- Formatting
- Best practices

---

## PEP 20 – The Zen of Python

https://peps.python.org/pep-0020/

A concise collection of guiding principles behind Python's design.

Example principles include:

- Simple is better than complex.
- Readability counts.
- Explicit is better than implicit.

---

## PEP 274 – Dict Comprehensions

https://peps.python.org/pep-0274/

Introduces dictionary comprehensions and explains their purpose.

Note: Dictionary comprehensions are beyond the scope of this module but become useful in later Python learning.

---

## PEP 3106 – Dictionary Views

https://peps.python.org/pep-3106/

Explains the behavior of dictionary view objects such as:

- keys()
- values()
- items()

Understanding these views helps explain how modern Python dictionaries work.
