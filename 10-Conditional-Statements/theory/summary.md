## Introduction

This module introduced **conditional statements**, one of the most fundamental concepts in Python programming. Conditional statements allow a program to make decisions by evaluating conditions and executing different blocks of code based on whether those conditions are `True` or `False`.

Without conditional statements, programs would execute every statement in the same order regardless of the situation. By using decision-making structures, programs become interactive, flexible, and capable of solving real-world problems.

# Conditional Statements Overview

Python provides several ways to make decisions.

| Statement | Purpose |
|-----------|---------|
| `if` | Execute code only when a condition is `True`. |
| `if-else` | Choose between two possible execution paths. |
| `if-elif-else` | Select one action from multiple possible outcomes. |
| Nested `if` | Make decisions in multiple stages. |

Each structure is useful in different situations.

---

# Decision-Making Process

Most conditional programs follow these steps:

1. Evaluate a condition.
2. Determine whether the condition is `True` or `False`.
3. Execute the appropriate block of code.
4. Continue with the remaining program.

This process is known as **decision making** or **control flow**.

---

# Boolean Expressions

Conditional statements rely on Boolean expressions.

Every condition evaluates to one of two values:

- `True`
- `False`

Boolean values are commonly produced using comparison operators.

Examples:

```python
age >= 18
marks == 50
temperature < 0
```

---

# Comparison Operators

You used comparison operators to compare values.

| Operator | Meaning |
|----------|---------|
| `==` | Equal to |
| `!=` | Not equal to |
| `>` | Greater than |
| `<` | Less than |
| `>=` | Greater than or equal to |
| `<=` | Less than or equal to |

These operators return Boolean values that can be used in conditional statements.

---

# Logical Operators

Logical operators combine multiple conditions.

| Operator | Purpose |
|----------|---------|
| `and` | All conditions must be `True`. |
| `or` | At least one condition must be `True`. |
| `not` | Reverses a Boolean value. |

These operators make conditional statements more powerful and expressive.

---

# Choosing the Right Conditional Statement

Use the appropriate statement for the problem you are solving.

| Situation | Recommended Statement |
|-----------|----------------------|
| One condition | `if` |
| Two possible outcomes | `if-else` |
| Multiple possible outcomes | `if-elif-else` |
| One decision depends on another | Nested `if` |

Choosing the correct structure improves readability and program organization.

# Best Practices Recap

When writing conditional statements:

- Use meaningful variable names.
- Keep conditions simple and readable.
- Maintain consistent indentation.
- Arrange conditions logically.
- Avoid unnecessary nesting.
- Test both `True` and `False` execution paths.
- Use logical operators carefully.
- Write code that is easy to understand.

These habits improve code quality and make programs easier to maintain.

---

# Common Mistakes Recap

Avoid these common errors:

- Forgetting the colon (`:`).
- Incorrect indentation.
- Using `=` instead of `==`.
- Placing conditions in the wrong order.
- Choosing the wrong logical operator.
- Writing overly complex conditions.
- Forgetting to test different input values.

Recognizing these mistakes will help you debug programs more effectively.
