# Arithmetic Operators at a Glance

| Operator | Name | Purpose | Example | Result |
|----------|------|---------|---------|--------|
| `+` | Addition | Adds values | `10 + 5` | `15` |
| `-` | Subtraction | Finds the difference | `10 - 5` | `5` |
| `*` | Multiplication | Multiplies values | `10 * 5` | `50` |
| `/` | Division | Performs true division | `10 / 5` | `2.0` |
| `//` | Floor Division | Returns the floor of the quotient | `10 // 3` | `3` |
| `%` | Modulus | Returns the remainder | `10 % 3` | `1` |
| `**` | Exponentiation | Raises a value to a power | `2 ** 3` | `8` |

# Operator Precedence

Python evaluates arithmetic expressions according to the following order:

| Priority | Operator |
|----------|----------|
| 1 | `()` |
| 2 | `**` |
| 3 | `*`, `/`, `//`, `%` |
| 4 | `+`, `-` |

Example:

```python
result = 10 + 4 * 3
print(result)
```

**Output**

```text
22
```

Using parentheses:

```python
result = (10 + 4) * 3
print(result)
```

**Output**

```text
42
```

# Common Mistakes to Remember

Avoid these common errors:

- Using `^` instead of `**` for exponentiation.
- Confusing `/` with `//`.
- Forgetting that `/` always returns a `float`.
- Misunderstanding the `%` operator.
- Ignoring operator precedence.
- Forgetting to convert user input to numeric types.
- Dividing by zero.
- Using unclear variable names.
- Writing overly complex arithmetic expressions without parentheses.
