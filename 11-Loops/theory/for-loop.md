# For Loop

## Introduction

A **`for` loop** is a control flow statement that repeatedly executes a block of code by iterating over a sequence of values.

Unlike a `while` loop, which continues as long as a condition is `True`, a `for` loop automatically retrieves each value from a sequence and executes the loop body once for each value.

In this module, the primary sequence you will use is the one generated by the `range()` function.

---

# Syntax

```python
for variable in sequence:
    statement(s)
```

Where:

- `for` is a Python keyword.
- `variable` receives one value from the sequence during each iteration.
- `in` is a Python keyword used to access values from a sequence.
- `sequence` is the collection of values being iterated over.
- The indented statements form the **loop body**.

---

# Flow of Execution

A `for` loop executes in the following order:

1. Retrieve the first value from the sequence.
2. Store it in the loop variable.
3. Execute the loop body.
4. Retrieve the next value.
5. Repeat until no values remain.
6. Exit the loop.

Flow diagram:

```text
          Start
            │
            ▼
   Get Next Value
            │
     ┌──────┴──────┐
     │             │
 Value Exists?     No
     │             │
    Yes            ▼
     │            End
     ▼
Execute Loop Body
     │
     └──────────────► Get Next Value
```

# Example 1: Print Numbers from 1 to 5

```python
for number in range(1, 6):
    print(number)
```

**Output**

```text
1
2
3
4
5
```

| 2 | 1 |

---

# `for` Loop vs `while` Loop

| `for` Loop | `while` Loop |
|------------|--------------|
| Iterates over a sequence | Repeats while a condition is `True` |
| Often used when the number of iterations is known | Often used when the number of iterations is unknown |
| Automatically retrieves the next value | Requires manual updates to the loop variable |
| Less likely to produce an infinite loop | Can easily become infinite if the condition never changes |

---

# Common Uses of `for` Loops

Use a `for` loop when:

- Printing a known range of numbers.
- Displaying a message a fixed number of times.
- Generating multiplication tables.
- Drawing simple patterns.
- Performing repeated calculations with a known number of iterations.

---

# Common Mistakes

## 1. Forgetting the Colon

Incorrect:

```python
for number in range(5)
    print(number)
```

Correct:

```python
for number in range(5):
    print(number)
```

---

## 2. Incorrect Indentation

Incorrect:

```python
for number in range(5):
print(number)
```

Result:

```text
IndentationError
```

---

## 3. Assuming `range(5)` Starts at 1

Incorrect expectation:

```text
1 2 3 4 5
```

Actual output:

```text
0 1 2 3 4
```

To start from `1`, write:

```python
range(1, 6)
```

---

# Best Practices

- Use meaningful loop variable names.
- Choose the appropriate form of `range()`.
- Keep the loop body simple and readable.
- Maintain consistent indentation.
- Use a `for` loop when the number of iterations is known or can be represented by a sequence.

---

# Key Points

- A `for` loop iterates over a sequence.
- The loop variable receives one value during each iteration.
- `range()` is the most common sequence used with `for` loops.
- A `for` loop automatically moves to the next value.
- A `for` loop may execute zero or more times, depending on the sequence.

---

# Summary

The `for` loop is the preferred choice when iterating over a known sequence of values. Combined with the `range()` function, it provides a concise and readable way to perform repetitive tasks without manually managing a loop condition or counter variable. Understanding the `for` loop is essential for writing efficient and maintainable Python programs.
